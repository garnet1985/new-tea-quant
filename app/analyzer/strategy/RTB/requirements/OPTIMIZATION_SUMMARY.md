# RTB 策略优化总结

## 概述

本文档记录了RTB（Rebound from Trough after Big drop）策略从初始设计到最终优化的完整过程。

**时间**：2025-10-11  
**方法**：机器学习 + 统计分析 + 数学优化  
**数据**：50只股票，2010-2024年，517笔交易  

---

## 最终成果

### V8 最终版本参数

**信号条件（6个过滤）**：
```python
ma_std < 0.03                    # 均线收敛
position < 0.30                  # 价格低位
drawdown_120d >= 0.20            # 前期大跌 ⭐ 最重要
0.009 < ma60slope < 0.176        # 趋势微涨
return_20d <= -0.07              # 近期回调
close_to_ma20 <= -0.05           # 低于均线
```

**止损止盈**：
```python
止损: -20%
止盈: 
  10%档: 0%  （继续持有）
  20%档: 50% （锁定一半）
  30%档: 50% （全部止盈）
```

**核心指标**：
- 期望值：**3.23%**（净期望：2.92%）
- 胜率：52.0%
- 20%达成率：35.2%
- 凯利仓位：14.6%
- 年化收益估算：**32.2%**

## 优化历程

### V5：基础版本

**参数**：
- 信号：ma_std + position + ma60slope（3个条件）
- 止盈：(30/30/40)
- 止损：-15%

**结果**：
- 信号数：668
- 期望值：1.42%
- 20%达成率：23.8%

**问题**：
- 期望值偏低
- 信号质量不高
- 需要更强的过滤

---

### V6：添加前期跌幅

**关键改进**：添加 `drawdown_120d >= 0.20`

**ML发现**：
- Feature Importance: 0.35（最高）
- 成功vs失败差异显著（T-test p<0.001）
- 前期跌幅是最关键的预测因子

**结果**：
- 信号数：1270
- 期望值：2.76%（**+94%**）
- 20%达成率：31.7%

**问题**：
- 止损率仍然偏高（32.7%）
- 信号质量仍有提升空间

---

### V7：统计优化

**关键改进**：
1. 添加 `return_20d <= -0.07`（买入前要下跌）
2. 添加 `close_to_ma20 <= -0.05`（价格低于MA20）
3. 优化 `ma60slope >= 0.009`（必须微涨，不能平稳）
4. 优化止损到 -20%
5. 取消保本止损

**ML统计分析**：
- 对比成功/失败交易的所有特征
- 使用T-test识别显著差异
- 确定最优阈值

**结果**：
- 信号数：517
- 期望值：2.85%（+3%）
- 20%达成率：33.8%
- 止损率：32.3%

**问题**：
- 分段止盈比例是否最优？

---

### V8：止盈优化（最终版）

**关键改进**：优化分段止盈比例

**方法**：
- 测试12种不同的分段止盈方案
- 计算每种方案的真实期望值
- 选择期望值最高的方案

**测试结果**：
| 方案 | 期望值 | 胜率 |
|-----|--------|------|
| 全仓(0/0/100) | 3.89% | 48.2% |
| 两段式(0/50/50) | **3.23%** | **52.0%** ✅ |
| 当前(30/30/40) | 2.83% | 54.7% |

**为什么选择(0/50/50)？**：
- 期望值高（3.23%）
- 胜率保持52%（过半）
- 在20-30%区间多持仓（主要收益来源）
- 平衡收益和风险

**最终结果**：
- 信号数：517
- 期望值：3.23%（**+13.3%**）
- 胜率：52.0%
- 20%达成率：35.2%
- **30%达成率：9.3%**（V7的9倍！）

## 关键发现

### 1. 前期跌幅最重要（drawdown_120d）

**Feature Importance**: 0.35

**数据**：
- 成功交易平均跌幅：39.6%
- 失败交易平均跌幅：31.6%
- 差异：8.0个百分点（显著）

**结论**：
- 跌得越多，反弹潜力越大
- 这是策略的核心逻辑

### 2. 趋势必须微涨，不能平稳（ma60slope）

**ML发现**：
- 成功交易 ma60slope：0.045（微涨）
- 失败交易 ma60slope：0.021（接近平）

**结论**：
- 完全平稳不是反转信号
- 必须有"微涨"迹象

### 3. 买入前要回调（return_20d）

**数据**：
- 成功交易 return_20d：-14.2%
- 失败交易 return_20d：-10.9%

**结论**：
- 买入前跌得更多的机会更好
- 避免追涨

### 4. 止损-20%优于-15%

**测试结果**：
- -15%止损：期望值 2.76%
- -20%止损：期望值 2.85%

**原因**：
- 止损过严增加止损次数
- 错失后续反弹机会
- -20%是最优平衡点

### 5. 保本止损有害

**分析结果**：
- 有保本止损：期望值 2.57%
- 无保本止损：期望值 2.85%

**原因**：
- 震荡市场频繁触发
- 失去后续上涨机会
- 降低整体收益

### 6. 在20%档多锁定更优

**核心Insight**：
- 20-30%区间是主要收益来源（32.9%的交易）
- 在20%锁定50%仓位
- 剩余50%继续搏30%
- 平衡收益和风险

## 方法论总结

### 1. 特征工程

从K线数据提取有意义的特征：
- 均线相关：ma_std, ma60slope, close_to_ma20
- 价格位置：position
- 历史表现：drawdown_120d, return_20d
- 波动性：volatility

### 2. 机器学习应用

不是用ML做预测，而是：
- **特征重要性分析**：找最关键的特征
- **统计显著性检验**：T-test找显著差异
- **参数优化**：网格搜索找最优阈值

### 3. 回测验证

- 信号去重（120天冷却期）
- 真实交易模拟（止损止盈）
- 多版本对比
- 期望值计算

### 4. 数学优化

- 期望值 = P(盈利) × 平均盈利 + P(亏损) × 平均亏损
- 盈亏比 = 平均盈利 / |平均亏损|
- 凯利准则：最优仓位计算

## 使用建议

### 仓位管理

**凯利建议**：14.6%
**保守建议**：7-8%

**分散投资**：
- 不要单一持仓
- 建议持有3-5个信号
- 总仓位不超过50%

### 执行纪律

1. **严格止损**：-20%必须离场
2. **分段止盈**：20%、30%按比例卖出
3. **不要主观判断**：相信系统
4. **记录每笔交易**：持续优化

### 监控指标

- 实际期望值 vs 理论期望值
- 胜率变化
- 止损率变化
- 信号质量
