# 页面文档：`setup.html`

## 这个页面是干什么的？
安装引导页（Setup Wizard），在系统未就绪时引导用户逐步完成初始化。

## 页面目的与用户价值
- 把复杂安装流程拆成少量可理解步骤，降低上手门槛。
- 用明确顺序确保必要配置完整，减少后续运行失败。
- 提供“安装完成”明确反馈，建立用户信心。

## 页面功能描述
- 分步向导流程（简洁、一步一屏）。
- 自动步骤自动执行并推进；需要用户输入的步骤停留等待。
- 关键步骤覆盖：
  - DB 配置检查/填写
  - 依赖安装
  - 初始数据导入
  - 完成页（第 5 页，独立 SUCCESS 页面）
- 页面不包含 header 导航和任何外链，避免用户跳转离开流程。

## 页面假设与 Placeholder
- 当前不接 launcher/install 实际状态机，步骤结果为前端模拟。
- 快捷入口为静态文本或占位，不跳外部链接。
- 失败重试、诊断日志仅作占位提示，待后续接 API/CLI 集成。
