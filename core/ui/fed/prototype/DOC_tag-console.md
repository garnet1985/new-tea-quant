# 页面文档：`tag-console.html`

## 这个页面是干什么的？
标签工作台，用于按 scenario 管理 tag 配置并触发标签运行。

## 页面目的与用户价值
- 把原本分散在配置文件中的标签设置可视化，降低维护难度。
- 通过“场景切换 + 分层配置”提高可读性，减少误改。
- 支持运行反馈，帮助用户验证配置是否有效。

## 页面功能描述
- 左右布局：
  - 左侧 scenario 列表，点击切换右侧配置。
  - 右侧配置区。
- 顶部 Scenario Meta 只读摘要（精简字段）。
- 两个 accordion：
  - 核心配置（默认展开）：场景信息、数据配置（target + contract 依赖）、tags 定义。
  - 其他配置（默认收起）：更新方式与性能等低频项。
- 支持 tags 运行动作，展示进度与运行结果文本。
- 移除“查看标签”入口（当前阶段不开放）。

## 页面假设与 Placeholder
- 场景数据和运行结果均为前端样例。
- 保存、运行不写后端，仅更新页面状态。
- 复杂字段（JSON、批量参数）采用简化输入模型占位。
