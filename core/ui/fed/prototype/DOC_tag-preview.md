# 页面文档：`tag-preview.html`

## 这个页面是干什么的？
查看标签结果的预览页（当前版本已降级为非主入口页面）。

## 页面目的与用户价值
- 用于探索“标签结果如何被查看”的可能交互形态。
- 验证按 scenario 过滤和核心字段展示是否满足基础排查诉求。

## 页面功能描述
- 页面标题为“查看标签”。
- 仅按 URL 传入 scenario 展示，不提供“全部场景”切换。
- 表格字段为：`entity_type`、`entity_id`、`tag_name`、`as_of_date`、`json_value`。
- 顶部统计卡和详情区随当前过滤结果变化。

## 页面假设与 Placeholder
- 当前阶段不作为正式入口（已从 tag-console 移除链接）。
- 数据量和分页问题未展开，数据为前端 mock。
- 后续是否保留取决于大数据量场景下的查询方案。
