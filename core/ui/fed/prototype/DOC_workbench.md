# 页面文档：`workbench.html`

## 这个页面是干什么的？
策略工作台首页，用于展示策略列表和进入策略调试流程（进入单个策略的验证页）。

## 页面目的与用户价值
- 给用户一个统一入口管理策略资产（查看、筛选、选择）。
- 快速进入“调试某个策略”的核心路径，减少导航成本。
- 作为后续扫描、回测、标签等功能的上游起点。

## 页面功能描述
- 展示策略列表（名称、状态、基础信息）。
- 提供进入策略验证/调试页的入口动作。
- 支持与全局导航联动（机会扫描、高级功能、设置）。
- 预留策略维度的常见操作入口（如后续扩展排序、筛选、搜索）。

## 页面假设与 Placeholder
- 当前为 UI Prototype，不依赖真实 API。
- 列表数据使用静态样例或前端内置 mock。
- 操作结果（如跳转、状态变化）以页面跳转和占位反馈为主。
