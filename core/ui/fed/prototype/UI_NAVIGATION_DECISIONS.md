# NTQ Prototype 导航决策记录

更新时间：2026-04-23（静态原型阶段小结）

## 背景

当前原型的目标是降低使用难度，并优先服务用户最常见的两类动作：

1. 验证策略想法（工作台内完成：列表 → 单策略验证）
2. 基于最新数据扫描机会（扫描）

同时保留专业能力入口，避免让专业用户找不到高级配置。

## 导航方案结论

主导航采用 4 项：

- 策略工作台（原「策略回测」入口；列表 + 单策略验证）
- 机会扫描
- 高级功能（可简称：高级）
- 设置

## 设计原则

- 核心能力前置：`策略工作台`、`机会扫描` 作为一线入口
- 专业能力收纳：将策略配置、标签配置、数据契约、数据抓取等归入 `高级功能`
- 低学习成本：普通用户路径最短，专业用户仅多一次点击
- 统一心智：避免在首页同时暴露过多专业术语

## 页面职责（与当前原型对齐）

### 策略工作台

- **列表页**（`workbench.html`）：展示全部策略与状态，点选进入单策略验证。
- **验证页**（`workbench-detail.html`）：双栏布局；主内容区适度加宽（约 `90vw` + 桌面最小宽度）。
  - **左侧**：策略 meta、Settings 预览、运行历史、其他策略切换。
  - **页顶显眼位置**展示当前实验使用的**回测数据区间**（占位常量；日后由配置/API 下发），与运行快照中的描述一致。
  - **右侧**：三层**执行计划**（枚举机会 → 价格回测 → 资金模拟）；每层内嵌占位结果字段，跑完后填入数字；三层逻辑依赖为：**仅枚举为底座**，价格与资金互不依赖（单独跑其中一层不应作废另一层）；未跑枚举时播放「价格 / 资金」会先自动补齐枚举。
  - **样本股票**：位于**右侧执行计划下方**（不再放在整页底部）；列表顺序默认与接口一致；视图内可排序/搜索。**在当前参数下三层均跑完后**才解锁样本表（占位逻辑如此，便于强调完整实验闭环）。
- **单股 K 线**（`workbench-stock.html`）：独立页；顶栏一行上下文（策略、标的、代码等）；三个 Tab 对应三层步骤名；下方 K 线占位 + 简要「接入后要画的标注」说明；返回验证页。

### 机会扫描

- 目标：即刻扫描市场机会。
- 重点：数据更新（可集成在扫描流程中）、扫描结果、机会分发。
- **说明**：与工作台的交叉导流、统一「实验/运行」身份等，留给 React 产品与 API 定义阶段细化。

### 高级功能

- 目标：管理扩展能力与专业配置。
- 建议分组（文案可再简化）：策略规则、市场标签、数据映射（数据契约）、数据来源、输出方式（适配器）。
- **说明**：静态原型阶段多为占位；与设置一同在正式实现时再充实。

### 设置

- 目标：系统级与用户级配置。
- 重点：账户偏好、运行参数、环境与连接配置。

## 页面文件命名（当前）

- `prototype/workbench.html`：策略工作台（列表）
- `prototype/workbench-detail.html`：单策略验证（双栏 + 右侧执行计划与样本股票）
- `prototype/workbench-stock.html`：样本股票 K 线（占位 Tab + 图层说明，独立入口）
- `prototype/backtest.html`：保留为跳转至 `workbench.html`（旧链接兼容）
- `prototype/scan.html`：机会扫描
- `prototype/advanced.html`：高级功能
- `prototype/settings.html`：设置

## 后续实施（React / API）

以下为已知留待正式栈处理的项，不改变当前导航与页面划分结论：

- 运行批次 / `run_id`、快照与历史、K 线图层与本页 Tab 的数据契约对齐。
- 工作台与机会扫描之间的入口与上下文传递。
- 禁用策略、接口失败等统一异常与文案。
- 无障碍、移动端与性能细节。

当前静态原型可作为信息架构与主路径的基准；细节在 React 落地与 API 定型时一并收敛即可。
